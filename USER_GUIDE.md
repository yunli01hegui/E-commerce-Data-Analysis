# 📘 电商全栈数据分析系统 - 用户使用指南

本系统旨在协助电商运营团队通过数据驱动决策。本指南将引导您快速上手数据大屏的可视化分析、数据中心的 CRUD 管理以及 AI 智能报告生成功能。

---

## 🚀 1. 快速上手：环境搭建

如果您是**开发者**，请参考以下步骤启动系统：

### 1.1 后端启动 (Python Flask)
1.  **安装 Python**: 确保安装了 Python 3.9 或以上版本。
2.  **安装依赖**: `pip install -r backend/requirements.txt`。
3.  **配置数据库**: 修改 `backend/app.py` 中的 `DB_PASSWORD` 及 MySQL 连接信息。
4.  **初始化**: 执行 `python backend/db_init.py` 以创建表并导入种子数据。
5.  **运行**: `python backend/app.py`（服务将运行在 `http://localhost:5000`）。

### 1.2 前端启动 (Vue 3)
1.  **安装 Node.js**: 确保安装了 Node.js 18 或以上版本。
2.  **安装依赖**: `npm install`。
3.  **运行**: `npm run dev`（访问 `http://localhost:5173`）。

## 📊 2. 数据大屏：可视化深度洞察

数据大屏分为 7 个模块，建议按以下逻辑进行分析：

### 2.1 销售趋势与 KPI
*   **如何看**: 关注顶部 KPI 卡片中的“月度增长率 (MoM)”。
*   **价值**: 识别日/月销量波动，发现大促（如双 11）带来的峰值。

### 2.2 RFM 客户模型 ⭐
*   **操作**: 观察“价值分布矩阵”中的气泡分布。
*   **策略**: 
    *   **重要价值客户**: 提供 VIP 专属服务，防止流失。
    *   **重要挽留客户**: 系统自动识别出的“曾经的高价值但近期未购买”群体，需立即推送优惠券激活。

### 2.3 智能推荐模块
*   **个性化推荐**: 选择任一活跃用户，系统将基于“协同过滤”算法生成专属商品清单。
*   **组合推荐**: 查看“关联规则”，发现常被打包购买的商品（如美妆+服装），辅助套餐设计。

## 🗄️ 3. 数据中心：跨平台 CSV 管理

数据中心是系统的“心脏”，支持全方位的订单管理。

### 3.1 跨平台数据上传
*   **功能**: 支持上传 **1688、京东、拼多多、淘宝、天猫** 的导出 CSV 数据。
*   **操作**: 点击右上角“上传 CSV”，选择文件。
*   **注意**: 系统会自动根据表头（如 `user_id`、`amount`）进行匹配。上传成功后，大屏数据将实时更新。

### 3.2 生产级 CRUD
*   **搜索**: 支持通过“所有项”或指定“用户ID/商品名/城市”进行模糊搜索。
*   **编辑**: 点击列表行内的“编辑”按钮，可直接修改价格、数量等字段。
*   **批量操作**: 勾选多行订单，可一键执行“批量删除”。
*   **清空**: “清除全部数据”按钮适用于重新导入全新平台数据。

## 🤖 4. AI 智能分析：深度决策专家

AI 页面连接了 **DeepSeek** 大语言模型。

### 4.1 报告类型
*   **数据分析报告**: 提供 GMV、客单价及各品类表现的宏观评估。
*   **用户行为分析**: 挖掘下单时段背后的消费心理，提出运营“助推 (Nudge)”方案。
*   **推荐优化建议**: 为技术团队提供算法迭代建议。

### 4.2 智能缓存逻辑
*   **说明**: 只要您的数据没有变动，AI 报告将瞬间从本地缓存加载（不消耗 Token）。
*   **更新**: 若您刚上传了新数据，点击“更新报告”可强制 AI 重新扫描数据库并生成最新洞察。

## 🔒 5. 常见问题 (FAQ)

*   **Q: 为什么 AI 报告生成失败？**
    *   A: 请检查 `backend/app.py` 中的 `DEEPSEEK_API_KEY` 是否配置正确，并确保网络能够连接到 DeepSeek 接口。
*   **Q: 上传 CSV 后大屏没变化？**
    *   A: 请确认 CSV 表头是否包含必填项（`user_id`, `amount`, `category` 等）。
*   **Q: 如何连接我自己的 MySQL 数据库？**
    *   A: 在 `backend/app.py` 中修改 `engine` 连接字符串即可。
