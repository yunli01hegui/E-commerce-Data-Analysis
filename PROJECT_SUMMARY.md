# 📖 电商全栈数据分析系统 - 技术说明文档

本项目是一套集数据采集、处理、可视化分析及 AI 决策支持于一体的电商全栈解决方案。系统采用现代化的前后端分离架构，通过高效的算法引擎为电商运营提供数据支撑。

---

## 一、 系统架构 (Architecture)

### 1. 技术栈概览
*   **前端 (Frontend)**: 
    *   **核心框架**: Vue 3.4 (Composition API) + TypeScript
    *   **可视化**: ECharts 5.5 + `vue-echarts` (提供高性能的交互式图表)
    *   **样式**: Tailwind CSS v4 + Lucide Vue Next (图标)
    *   **路由/构建**: Vue Router + Vite
*   **后端 (Backend)**:
    *   **核心框架**: Python Flask (轻量级 RESTful API)
    *   **数据处理**: Pandas (负责大规模订单数据的向量化计算与统计)
    *   **ORM/数据库**: SQLAlchemy + MySQL 8.0 (持久化存储)
    *   **API 文档**: Flasgger (Swagger UI)
*   **AI 层**:
    *   **模型**: DeepSeek-V3 API
    *   **协议**: OpenAI 兼容协议 (Chat Completions)

## 二、 核心算法逻辑

### 1. RFM 客户价值模型
系统基于 **R (Recency/最近消费天数)**、**F (Frequency/消费频次)**、**M (Monetary/消费总金额)** 对用户进行 5 分位评分（1-5 分）。
*   **计算逻辑**: 使用 `pd.qcut` 对用户 R/F/M 指标进行分段。
*   **分群规则**: 根据评分组合（如 R>=4, F>=4, M>=4 为重要价值客户）自动将用户划分为 8 个精细化运营群体。

### 2. 智能推荐引擎
*   **基于用户的协同过滤 (User-Based CF)**: 
    *   构建“用户-品类”消费矩阵。
    *   使用 **余弦相似度 (Cosine Similarity)** 计算用户间的品类偏好距离。
    *   推荐逻辑：找到最相似的 10 位邻居，推荐当前用户未购买但邻居高频购买的 Top 6 商品。
*   **月度关联规则挖掘**: 
    *   分析同一用户在自然月内的“购物篮”组合。
    *   挖掘跨品类关联（如：买过“电子产品”的人常在同月购买“美妆”）。

## 三、 AI 报告生成机制

### 1. 数据库版本校验 (Versioning)
为了降低 API 调用成本并提升响应速度，系统实现了基于数据库指纹的缓存机制：
*   **检测**: 每次请求 AI 报告前，后端执行 `CHECKSUM TABLE orders`。
*   **逻辑**: 只有当数据库发生增删改操作（指纹改变）时，才会重新调用 DeepSeek API；否则直接从 `ai_cache.json` 读取缓存内容。

### 2. 结构化报告渲染
AI 报告支持 Markdown 格式，前端通过自定义解析器实时渲染多级标题、表格、代码块及加粗强调。

## 四、 数据中心 (Data Center)

### 1. 跨平台支持
系统内置了对 **1688、京东、拼多多、淘宝、天猫** 等主流平台 CSV 导出格式的兼容处理。
*   **导入逻辑**: 上传 CSV 后，后端通过 Pandas 清洗无效数据，并利用 `to_sql` 增量或全量同步至 MySQL。

### 2. 生产级管理功能
*   **列表**: 支持对数千条订单进行分页、关键词全局模糊搜索及特定字段筛选。
*   **CRUD**: 支持手动新增订单、行内实时修改数据及批量删除。

---

## 五、 开发与部署建议

*   **性能优化**: 后端 Pandas 计算已进行向量化处理，支持万级数据的秒级统计。
*   **扩展性**: 推荐算法接口采用插拔式设计，可轻松更换为协同过滤或深度学习模型。
*   **安全性**: 建议生产环境下对 Flask 接口进行 Token 验证，并对 MySQL 数据库开启索引优化（特别是 `user_id` 和 `purchase_time`）。
