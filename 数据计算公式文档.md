# 📊 电商数据分析系统 - 核心算法与 KPI 计算文档

本文档详细说明了系统中各数据模块的底层计算公式。所有逻辑均在后端 Python (Pandas + SQLAlchemy) 中实现，确保了统计的精确性与高性能。

---

## 1. 核心 KPI 指标 (Global Stats)

### 1.1 总销售额 (GMV)
*   **公式**: $GMV = \sum Amount_i$
*   **逻辑**: 对数据库中所有订单记录的 `amount` 字段进行求和。

### 1.2 平均客单价 (AOV)
*   **公式**: $AOV = GMV / TotalOrders$
*   **逻辑**: 总销售额除以订单总数。

### 1.3 月度环比增长率 (MoM Growth)
*   **公式**: $Growth = \frac{CurrentMonthSales - LastMonthSales}{LastMonthSales} \times 100\%$
*   **逻辑**: 对比最新月份与上一个自然月的销售总额。若上月无数据，则默认为 0。

---

## 2. RFM 客户价值模型算法 ⭐

系统使用 **五分位数法 (Quantile-based Score)** 对用户进行评分。

### 2.1 三大维度定义
*   **R (Recency/最近消费天数)**: 当前日期减去用户最后一次下单日期（天数越少，R 评分越高）。
*   **F (Frequency/消费频次)**: 用户在统计期内的总订单量（次数越多，F 评分越高）。
*   **M (Monetary/消费总金额)**: 用户在统计期内的总消费金额（金额越高，M 评分越高）。

### 2.2 评分与分群
*   **评分**: 利用 `pd.qcut` 将 R/F/M 分别划分为 5 个等级 (1-5 分)。
*   **分群规则示例**:
    *   **重要价值客户**: $R \ge 4, F \ge 4, M \ge 4$。
    *   **重要挽留客户**: $R \le 2, F \le 2, M \ge 4$（即高消费潜力但长期未下单）。

## 3. 智能推荐算法逻辑

### 3.1 基于用户的协同过滤 (User-CF)
*   **向量构建**: 以用户 ID 为行，商品品类为列，建立“品类消费金额矩阵”。
*   **相似度计算 (余弦相似度)**:
    $$Similarity(U, V) = \frac{U \cdot V}{\|U\| \cdot \|V\|}$$
*   **推荐得分**: 相似邻居的偏好分值乘以相似度权重 $\sum (Sim_{UV} \times Preference_{Vi})$。

### 3.2 月度跨品类关联规则 (Association)
*   **支持度 (Support)**: $\frac{count(CategoryA \cap CategoryB)}{TotalMonthlyBaskets}$。
*   **逻辑**: 挖掘同一用户在同一个月内共同购买的不同品类配对。

---

## 4. 统计模块逻辑

### 4.1 年龄分布分析
*   **分段定义**: 划分为 7 个阶段：`0-18`, `18-25`, `26-35`, `36-45`, `46-55`, `56-65`, `66+`。
*   **核心群体识别**: 自动识别 `totalAmount` 最高的年龄段为“核心消费群体”。

### 4.2 城市分级逻辑
*   **一线城市**: `北京`, `上海`, `广州`, `深圳`。
*   **新一线城市**: `杭州`, `成都`, `重庆`, `南京`, `苏州`, `武汉`, `西安`, `天津`。
*   **其他**: 除上述城市外的所有地域。

---

## 5. 数据处理流程 (ETL)

1.  **加载**: 使用 Pandas 的 `read_sql` 从 MySQL 提取全量数据。
2.  **转换**: 强制转换 `purchase_time` 为 `datetime64[ns]` 类型以支持时间序列聚合。
3.  **计算**: 利用 `groupby` 进行分组统计，聚合结果通过 `to_dict('records')` 转化为 JSON 返回给前端。
4.  **同步**: 前端 `fetchAllStats` 通过 `Promise.allSettled` 并行请求所有接口，实现大屏状态的原子性刷新。
