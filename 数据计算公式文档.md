# 电商数据可视化分析系统 - 数据计算公式文档

## 一、原始数据格式（CSV输入）

### 订单数据字段
```
userId        - 用户ID
userName      - 用户姓名
productId     - 商品ID
productName   - 商品名称
category      - 商品品类（电子产品、服装、美妆、运动、书籍、玩具、家居）
price         - 商品单价
purchaseTime  - 购买时间（格式：2024/M/D H:M）
quantity      - 购买数量
amount        - 订单金额（price × quantity）
city          - 城市
gender        - 性别（男/女）
age           - 年龄
```

## 二、各模块数据计算公式

### 1. 商品分析（ProductAnalysis）

#### 1.1 品类销售占比
**输入**: 所有订单数据  
**计算逻辑**:
```javascript
// 按品类分组统计
categoryData = {
  name: category,          // 品类名称
  value: sum(amount),      // 该品类总销售额
  count: 订单数量
}

// 按 value 降序排序
```

**输出格式**:
```json
[
  { "name": "电子产品", "value": 销售额, "count": 订单数 },
  { "name": "服装", "value": 销售额, "count": 订单数 },
  ...
]
```

#### 1.2 热销商品排行（TOP 15）
**输入**: 所有订单数据  
**计算逻辑**:
```javascript
// 按商品名称+品类组合统计（同名不同品类视为不同商品）
productRanking = {
  key: `${productName}-${category}`,
  name: productName,       // 商品名称
  category: category,      // 所属品类
  sales: sum(quantity),    // 总销量
  revenue: sum(amount),    // 总销售额
  orders: 订单数量
}

// 按 revenue 降序排序，取前15条
```

**输出格式**:
```json
[
  {
    "name": "智能手机",
    "category": "电子产品",
    "sales": 总销量,
    "revenue": 总销售额,
    "orders": 订单数
  },
  ...

```

### 2. 销售趋势（SalesTrend）

#### 2.1 月度销售趋势
**输入**: 所有订单数据  
**计算逻辑**:
```javascript
// 从purchaseTime中提取月份（1-12月）
// 按月统计
monthlyData = {
  month: `${month}月`,              // 月份标签
  sales: sum(amount),               // 当月总销售额
  orders: count,                    // 当月订单数
  avgAmount: sales / orders         // 当月平均客单价
}
```

**输出格式**:
```json
[
  { "month": "1月", "sales": 销售额, "orders": 订单数, "avgAmount": 客单价 },
  { "month": "2月", "sales": 销售额, "orders": 订单数, "avgAmount": 客单价 },
  ...
  { "month": "12月", "sales": 销售额, "orders": 订单数, "avgAmount": 客单价 }
]
```

#### 3.2 日销售趋势（最近30天）
**输入**: 所有订单数据  
**计算逻辑**:
```javascript
// 从purchaseTime中提取日期（格式：2024/M/D）
// 按日期分组统计
dailyData = {
  date: 日期（去掉年份）,          // 如 "1/15"
  sales: sum(amount),              // 当日总销售额
  orders: count                    // 当日订单数
}

// 按日期排序，取最近30天
```

**输出格式**:
```json
[
  { "date": "11/15", "sales": 销售额, "orders": 订单数 },
  { "date": "11/16", "sales": 销售额, "orders": 订单数 },
  ...
]
```

#### 2.3 关键指标
**计算公式**:
```javascript
总销售额 = sum(所有订单的amount)
订单总量 = count(所有订单)
平均客单价 = 总销售额 / 订单总量
月度增长率 = (本月销售额 - 上月销售额) / 上月销售额 × 100%
```

### 6. RFM客户价值分析（RFMAnalysis）

#### 6.1 用户RFM指标计算
**参考日期**: 2024/12/31（或当前日期）

**第一步：按用户聚合订单数据**
```javascript
userRFM = {
  userId: 用户ID,
  userName: 用户名称,
  lastPurchaseDate: max(purchaseTime),   // 最后购买日期
  orderCount: count,                     // 订单数
  totalAmount: sum(amount),              // 总消费金额
  orders: [订单列表]
}
```

**第二步：计算RFM原始值**
```javascript
rfmData = {
  userId: 用户ID,
  userName: 用户名称,
  recency: 距参考日期的天数 = (参考日期 - lastPurchaseDate) / (1000*60*60*24),
  frequency: orderCount,                 // 购买频率
  monetary: totalAmount                  // 消费金额
}
```

#### 6.2 RFM评分（1-5分）
**评分算法**（五分位数法）:
```javascript
function getScore(value, allValues, reverse) {
  // 1. 将所有值排序
  sorted = reverse ? allValues.sort(降序) : allValues.sort(升序)
  
  // 2. 计算五分位大小
  quintile = sorted.length / 5
  
  // 3. 找到当前值的位置
  index = sorted.indexOf(value)
  
  // 4. 计算评分（1-5分）
  score = min(5, floor(index / quintile) + 1)
  
  return score
}

// 计算各维度评分
R评分 = getScore(recency, allRecencyValues, reverse=true)   // 最近越好，反向排序
F评分 = getScore(frequency, allFrequencyValues, reverse=false)
M评分 = getScore(monetary, allMonetaryValues, reverse=false)

RFM总分 = R + F + M  // 范围：3-15分
```

#### 6.3 客户分群规则
**分群逻辑**:
```javascript
if (R >= 4 && F >= 4 && M >= 4) → 重要价值客户
if (R >= 4 && F <= 2 && M >= 4) → 重要发展客户
if (R <= 2 && F >= 4 && M >= 4) → 重要保持客户
if (R <= 2 && F <= 2 && M >= 4) → 重要挽留客户
if (R >= 4 && F >= 4 && M <= 2) → 一般价值客户
if (R >= 4 && F <= 2 && M <= 2) → 一般发展客户
if (R <= 2 && F >= 4 && M <= 2) → 一般保持客户
if (R <= 2 && F <= 2 && M <= 2) → 一般挽留客户
else → 潜力客户
```

**RFM评分后数据格式**:
```javascript
rfmSegmented = {
  userId: 用户ID,
  userName: 用户名称,
  recency: 距今天数,
  frequency: 购买次数,
  monetary: 总消费金额,
  R: R评分（1-5）,
  F: F评分（1-5）,
  M: M评分（1-5）,
  rfmScore: R + F + M,
  segment: 客户分群标签,
  segmentColor: 分群颜色,
  segmentDesc: 分群描述
}

// 按rfmScore降序排序
```

#### 7.4 客户分群统计
**计算逻辑**:
```javascript
segmentStats = {
  segment: 分群名称,
  color: 分群颜色,
  desc: 分群描述,
  count: 该分群用户数,
  totalAmount: 该分群总消费金额

```

**计算逻辑**:
```javascript

  x: frequency,           // X轴：购买频率
```

## 四、数据处理流程

### CSV导入流程
```
1. 读取CSV文件
2. 解析每一行数据
3. 验证数据格式：
   - userId, productId 必填
   - amount = price × quantity
   - purchaseTime 格式验证
   - age 范围验证（0-100）
   - gender 枚举验证（男/女）
   - category 枚举验证
4. 数据清洗：
   - 去除重复订单
   - 处理缺失值
   - 异常值检测
5. 存储到数据库
```

### 数据更新策略
```
- 增量更新：只处理新增订单
- 全量更新：重新计算所有统计数据
- 建议：T+1日更新（每日凌晨批量计算）
```

## 五、SQL查询示例（参考）

### 性别分布查询
```sql
SELECT 
  gender AS name,
  COUNT(*) AS value,
  SUM(amount) AS amount
FROM orders
GROUP BY gender;
```

### 月度销售趋势
```sql
SELECT 
  MONTH(purchase_time) AS month,
  SUM(amount) AS sales,
  COUNT(*) AS orders,
  AVG(amount) AS avg_amount
FROM orders
WHERE YEAR(purchase_time) = 2024
GROUP BY MONTH(purchase_time)
ORDER BY month;
```

### RFM分析基础查询
```sql
SELECT 
  user_id,
  user_name,
  DATEDIFF('2024-12-31', MAX(purchase_time)) AS recency,
  COUNT(*) AS frequency,
  SUM(amount) AS monetary
FROM orders
GROUP BY user_id, user_name;
```

## 六、注意事项

1. **日期处理**
   - 前端使用的日期格式：`2024/M/D H:M`
   - 后端建议统一为标准时间戳或ISO格式

2. **数值精度**
   - 金额保留2位小数
   - 百分比保留1位小数

3. **性能优化**
   - 大数据量时使用分页
   - 预计算常用指标
   - 使用数据库索引（userId, purchaseTime, category, city）

4. **数据一致性**
   - amount = price × quantity
   - 用户去重使用 userId（不是 userName）

5. **边界情况处理**
   - 除数为0的情况（如某性别无订单）
   - 空数据集的处理
   - 日期范围边界
